# 知源 KnowSource — 项目目标

> 本文档是项目唯一维护的目标说明，供 AI 助手和开发者必读。

---

## 项目定位

知源是一个面向科研人员和知识工作者的桌面端文献阅读与知识管理工具。核心目标是帮助用户围绕自己上传的文献，建立可溯源、可交互的个人知识网络。

项目愿景：让每一份文献都有迹可循——阅读不再停留于零散地记忆结论，而是能回到论点在原文中的位置，理解概念之间的关联，并在灵感迸发时将其与知识点快速关联。

与通用文献管理工具的区别：解析结论默认锚定用户上传的原始文献。自动抽取的每个概念都能回溯到具体文档和位置；匹配结果可回溯到文档与章节；用户手动创建的节点或 Idea 可选择是否锚定。

---

## 目标用户

- **科研人员**：阅读大量论文、综述、技术报告，需要快速提取关键概念、建立跨文献知识网络
- **研究生/博士生**：整理课题文献，构建研究方向的知识图谱，写论文时快速定位引用来源
- **跨学科研究者**：将不同来源、不同格式的文献统一导入，发现领域之间的概念关联
- **知识工作者**：处理报告、方案、案例等多份文档，用图谱整理思路，用 Idea 记录洞察

典型使用流程：创建研究项目 → 批量导入文献 → 自动解析并生成知识图谱 → 浏览跨文献关联 → 双击节点跳转原文位置 → 在 Idea 面板记录灵感并与节点关联 → 切换项目独立管理不同课题。

## 核心功能目标

### 多格式文献解析

支持 PDF、Word(docx)、PPTX、Markdown、图片（OCR）的导入与解析。PDF 兼容文本层和扫描版（OCR 识别），解析过程中检测多栏排版、过滤页眉页脚、修复重影字符。所有格式的解析结果保留标题层级和位置信息，为后续溯源提供基础。

### 知识图谱构建

从文献中自动抽取概念和关系，构建可交互的力导向图谱。提供两条建图路径：无监督路径（TF-IDF、TextRank、共现统计、PageRank、社区发现）和 LLM 增强路径。LLM 不可用或超时时自动回退到无监督路径，保证核心流程不中断。支持概念合并、别名归一、跨文档连接、节点/边增删。

### LLM 处理透明化与自定义

所有 16 个 LLM 任务的提示词模板以静态字符串形式集中存储在 `core/prompts/templates.js` 中，使用 `{{变量名}}` 占位符语法。用户打开提示词实验室（PromptLab）即可立即看到全部内置模板，无需等待任务实际执行。用户直接编辑模板原文并保存，修改即生效；可随时恢复内置模板。

运行流程透明化：`runLLMTask` 从注册表取有效模板（用户覆盖优先，否则内置），用调用方传入的变量值渲染占位符，将渲染后的完整提示词发送给 LLM。每次调用（包括 PromptLab 中的手动测试）都会写入调用日志，记录渲染后的 system/user prompt、模型响应、耗时、成功状态、是否使用自定义模板。用户可在日志区展开查看模板填入实际内容后的完整提示词，便于调试和优化。

### 文献溯源

图谱节点保留来源文档与位置信息。用户在图谱中双击节点，确认后跳转到原文对应位置。匹配结果可定位到文档与章节/页面级别。Idea 引用支持关联到文档和节点。

### 多策略匹配

文献中的概念、问题或摘录通过四层策略与已有文献关联：关键词重叠（Jaccard）、TF-IDF 向量余弦相似度、语义 embedding 相似度、图结构匹配（共同邻居、PPR）。最终融合多层分数，给出跨文献的最优对应关系。

### Idea 管理

用户可以记录阅读时产生的灵感、问题、评论或研究假设，形成 Idea。Idea 可以与图谱节点关联，系统根据内容自动推荐相关节点。Idea 同步到图谱后作为节点存在，与文献节点、概念节点共同构成个人化的研究笔记网络。

### 多项目管理

每个课题或研究方向对应一个独立项目。项目之间数据隔离，支持创建、重命名、删除、导入、导出和切换。数据持久化到应用用户目录。

### 模型配置

支持本地模型（Ollama，可自动检测、拉取模型）、OpenAI 兼容的云服务接口（OpenAI、DeepSeek、SiliconFlow、OpenRouter、Moonshot、通义千问、智谱等）、HuggingFace 接口、stub 模式（无 LLM 降级）。核心 LLM 调用遵循失败回退原则，保证核心流程不中断。

---

## 桌面应用形态

知源是 Windows 桌面端软件，以 Electron + Vue 3 构建。所有功能集成在安装包中，用户安装后可直接使用，无需额外配置环境。开发环境通过 Vite 热更新，生产环境加载打包后的前端资源。后端服务内嵌于 Electron 主进程，通过 IPC 或 HTTP 通信。

---

## 代码结构

- `electron/`：主进程、预加载脚本、窗口管理、IPC 通道
- `renderer/src/`：Vue 3 前端（组件、状态管理、API 客户端、样式）
- `services/`：后端服务（API 路由、处理器、存储、LLM 适配、解析调度）
- `core/`：纯逻辑引擎（解析器、抽取器、图谱、匹配器、Idea 模型、管线编排、提示词系统）
  - `core/prompts/templates.js`：16 个 LLM 任务的内置提示词模板（静态字符串，`{{var}}` 占位符）
  - `core/prompts/registry.js`：提示词注册表，负责模板解析、用户覆盖管理、占位符渲染
  - `core/prompts/run-task.js`：统一 LLM 调用入口，从注册表取有效模板渲染后调用 provider
  - `core/prompts/logger.js`：LLM 调用日志环形缓冲区
- `tests/`：Node 原生测试
- `PROJECT_GOALS.md`：本文件（项目唯一维护的目标文档）
